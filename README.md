# Discord музыкальный бот

Бот на **discord.py v2.x** со **слэш-командами** (`/`): очередь треков на каждый
сервер, поиск и плейлисты YouTube, плейлисты Spotify, кнопки управления и
авто-выход. Звук — через **FFmpeg**, источники — **yt-dlp**.

## Команды
| Команда | Что делает |
|---|---|
| `/play <ссылка/название>` | Добавить трек в очередь и играть |
| `/queue` | Показать очередь |
| `/skip` | Пропустить текущий трек |
| `/pause` | Пауза |
| `/resume` | Продолжить |
| `/stop` | Остановить, очистить очередь и выйти из канала |

Что можно передавать в `/play`:
- **ссылку на YouTube** (и сотни других сайтов — всё, что понимает yt-dlp);
- **ссылку на плейлист YouTube** — все треки добавятся в очередь;
- **ссылку на Spotify** (трек / плейлист / альбом) — треки берутся из Spotify,
  а само аудио ищется и играет с YouTube (ключи и Premium не нужны);
- **прямую ссылку** на аудиофайл/поток (`https://example.com/audio.mp3`);
- **название трека словами** — бот найдёт первый результат на YouTube
  (например, `/play Король и Шут Лесник`).

## Кнопки под треком
При старте каждого трека бот шлёт сообщение **«Сейчас играет»** (название,
длительность, ссылка, превью, кто добавил) с кнопками: **⏸️ Пауза/Продолжить**,
**⏭️ Пропустить**, **⏹️ Стоп**. Управлять может любой участник — ограничений по ролям нет.

## Авто-выход
Бот сам выходит из канала через **2 минуты**, если очередь закончилась или в
канале не осталось людей.

## Почему не заикается
- **FFmpeg** (переподключение при любых сетевых проблемах CDN — главная причина рывков):
  `-reconnect 1 -reconnect_streamed 1 -reconnect_on_network_error 1 -reconnect_on_http_error 4xx,5xx -reconnect_delay_max 5`
- **yt-dlp**: формат `bestaudio/best` — берётся готовая аудиодорожка без лишнего
  перекодирования. Ссылка на поток резолвится прямо перед каждым треком, поэтому
  не «протухает» в длинной очереди.

## Что нужно установить

### 1. Python-библиотеки
```
pip install -r requirements.txt
```
Ставятся: `discord.py[voice]` (сам бот + голос: `PyNaCl` и `davey`), `yt-dlp`
(аудио из YouTube и др.), `requests` (чтение треков из Spotify) и `python-dotenv`.

### 2. FFmpeg (в системе)
Бот стримит звук через FFmpeg — он должен быть установлен.
- Проверка: в терминале `ffmpeg -version` — если выводит версию, всё ок.
- Если нет — установить (например, `winget install Gyan.FFmpeg`) и перезапустить терминал.
- Либо положить `ffmpeg.exe` в папку `ffmpeg\bin\` рядом с ботом — бот найдёт его сам.

## Настройка бота на стороне Discord
1. На https://discord.com/developers/applications создай приложение и бота.
2. Скопируй токен (**Reset Token**) и вставь его в файл `.env`:
   ```
   DISCORD_TOKEN=твой_токен
   ```
3. В разделе **Bot** оставь включённым **Message Content Intent** (он уже включён;
   без него discord.py пишет предупреждение в консоль при старте).
4. Пригласи бота на сервер (OAuth2 → URL Generator → scopes: `bot` **и**
   `applications.commands`; права: `Connect`, `Speak`, `Send Messages`, `Embed Links`).

## Spotify
Никакой настройки не нужно — **ни ключей, ни Premium**. Бот читает список треков
с публичной embed-страницы Spotify (`open.spotify.com/embed/...`) и проигрывает
их аудио с YouTube. Просто кидай ссылку:
```
/play https://open.spotify.com/playlist/...
/play https://open.spotify.com/album/...
/play https://open.spotify.com/track/...
```
Важно: это работает только для **публичных** плейлистов/альбомов. Аудио берётся
с YouTube, поэтому версия может чуть отличаться от оригинала.

## Запуск
```
python bot.py
```
После «✅ Бот запущен как ...» команды `/` появятся в Discord автоматически
(синхронизируются при старте). Зайди в голосовой канал и набери `/play`.
```
/play https://www.youtube.com/watch?v=...
```
